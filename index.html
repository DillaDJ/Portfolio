About me!

Thanks for taking a moment to visit my website, hopefully you like what you see! I was born in South Africa and I lived there until 2011, which is when my family and I moved to New Zealand. I've been programming since I was 10 years old and I have a keen interest in graphics and software in general, I think a long term goal of mine would be to work my way into becoming a technical artist but the learning curve is very steep. I've been keeping my learning in the technical art and gamedev field as a hobby and maybe someday it'll be a carreer option, but for now I'm going to keep learning.

I studied Computer Science and Linguistics at the University of Auckland (2017 - 2022), I know it's quite the mix but I found Linguistics interesting so I did it on the side, it's great fun. I don't mind getting my hands dirty, that is to say, learning new things and adding them to my toolkit as a developer. Which is why I'll jump around and make projects in all sorts of engines, frameworks and programming languages. I'd like think I pick up new things quickly because I have a base understanding of common concepts between all of them and pick everything else up as I go. I think that's about it for me, hope you think the things on this page are interesting!

I am skilled in:

Godot

Godot is one of the more recent engines I've been learning to use but I really like how lightweight and easy to use it is. I made a small multiplayer prototype card game in Godot 3.4 that's based off of a game called 'president' (or scum or tycoon, whatever you call it) that I'd like to port it to Godot 4 and use the new networking features. The language Godot uses, GDScript, is very similar to Python with its syntax and like Python is very easy to pick up and use.

I'd like to participate in a gamejam using Godot soonish and add it to this page, in the mean time I think I'll just port the card game to Godot 4, polish it a little and put it here too.

Unity

Unity is my most used engine by far, I started learning how to use it in 2016 and it's come a long way since then. Unity uses C# as it's programming language of choice and it is the language I am the most fond of. I have participated in one game jam for Unity (of which you can see and even play the submitted game in the work section) and made many prototypes over the years ranging from very simple point n clicks to training Machine Learning Agents to act as AI units in the game.

While Godot's node based model is growing on me, I am still most at home using gameobject although I would like to learn ECS when it is production ready.

Unreal Engine

Most of my endevours in Unreal Engine were in 2021, it is very feature packed which makes it feel cluttered at times but it looks fantastic out the box. It has a pretty steep learning curve and uses a combination of C++ and 'Blueprints' which is their name for visual scripting and the best workflow appears to be mixing and matching using the two but I mostly like to stick to the code and only use Blueprint if I have to. I have made a few prototypes in Unreal engine and a tetris clone.

I would like to continue my learning in Unreal Engine, especially with the introduction of Lumen and Nanite as flagship technologies
